Context de la pràctica

Per a quin motiu és rellevant aquesta pràctica?

En el món professional, és molt poc habitual que un servidor físic executi directament un sol sistema operatiu. La norma és virtualitzar: un únic servidor físic allotja diverses màquines virtuals que ofereixen serveis independents. Proxmox VE és un dels hipervisors de codi obert més usats en empreses i centres de dades.

Maquinari real

Gestió web

Proxmox VE ofereix una interfície web per gestionar totes les VMs des de qualsevol navegador a la mateixa xarxa. No cal cap client especial: és el paradigma dels centres de dades moderns.

Infraestructura completa

Al final de la pràctica, el grup tindrà una infraestructura funcional: un hipervisor amb un servidor de serveis (DNS, DHCP) i un client que valida que els serveis funcionen correctament.

Objectiu de la pràctica

Preparar el maquinari físic del taller, instal·lar i configurar Proxmox VE com a hipervisor tipus 1, desplegar una VM amb Zentyal que ofereixi serveis DNS i DHCP, i verificar-los amb una VM client Linux lleugera que ha de rebre IP exclusivament de Zentyal (no de la xarxa del centre).

Conceptes teòrics previs

El que has de comprendre abans de començar

Entendre l'arquitectura de virtualització és clau per diagnosticar problemes i prendre decisions durant la configuració. No és suficient seguir passos: has de saber per a quin motiu fas cada cosa.

Tipus d'hipervisors

Tipus 1 vs. Tipus 2: una diferència fonamental

Tipus 1

Bare-Metal (Proxmox, ESXi, Hyper-V)

S'instal·la directament sobre el maquinari físic, sense cap SO intermig. Té accés directe als recursos (CPU, RAM, discs, xarxa). Més rendiment, més eficient, usat en producció. El maquinari és dedicat exclusivament a la virtualització.

Exemples de producció

VMware ESXi, Microsoft Hyper-V Server, Proxmox VE, Xen. Tots gestionen des de centenars fins a milers de VMs.

Tipus 2

Hosted (VirtualBox, VMware Workstation)

S'executa sobre un SO hoste (Windows, Linux, macOS). Les VMs competeixen per recursos amb el SO base i les seves aplicacions. Rendiment menor, però molt més còmode per a ús personal i proves. El que has usat fins ara.

Analogia

Tipus 1 és el propietari de l'edifici. Tipus 2 és un llogater que subarrega habitacions als seus convidats.

Proxmox VE: arquitectura interna

Base Debian Linux

Proxmox VE és un sistema complet basat en Debian amb el kernel de Linux modificat per optimitzar la virtualització. La interfície web és una capa per sobre d'un sistema Linux completament funcional al qual pots accedir per SSH.

KVM + LXC

Proxmox suporta dues tecnologies: KVM (Kernel-based Virtual Machine) per a VMs completes amb SO independent, i LXC (Linux Containers) per a contenidors que comparteixen el kernel del host. En aquesta pràctica usem KVM.

Bridge de xarxa (vmbr0)

Proxmox crea un switch virtual (vmbr0) que connecta les VMs entre elles i cap a la xarxa física. Les VMs usen aquest bridge com si fos un cable de xarxa virtual. És essencial configurar-lo correctament perquè les VMs tinguin connectivitat.

Repositoris: Enterprise vs. No-Subscription

Per defecte, Proxmox configura el repositori Enterprise (requereix subscripció de pagament). Sense subscripció, cal afegir el repositori No-Subscription per poder actualitzar i instal·lar paquets. Sense fer-ho, les actualitzacions fallaran.

Emmagatzematge (Storage)

Proxmox gestiona l'emmagatzematge en "pools". Per defecte, crea un pool local on es guarden les ISOs i les imatges de disc de les VMs. En aquesta pràctica, l'emmagatzematge local és suficient.

Certificat autosignat

La interfície web de Proxmox (port 8006) usa HTTPS amb un certificat autosignat. El navegador mostrarà un avís de seguretat. Pots ignorar-lo per accedir (Avançat → Continua): en un entorn de producció real, s'usaria un certificat vàlid.

Zentyal i els serveis DNS/DHCP

Zentyal: servidor de serveis per a PIMES

Zentyal és una distribució Linux (basada en Ubuntu) dissenyada per ser un servidor de xarxa complet per a petites i mitjanes empreses. Ofereix una interfície web amigable per gestionar: DHCP, DNS, Firewall, Samba (Active Directory), Correu, VPN. En aquesta pràctica el fem servir per DNS i DHCP.

DNS i DHCP: per a quin motiu són claus?

DHCP assigna automàticament adreces IP als dispositius de la xarxa. Sense ell, cada dispositiu necessita configuració manual. DNS tradueix noms de domini (servidor.empresa.local) en adreces IP. Sense ell, els usuaris haurien de recordar IPs. Junts, formen la base de qualsevol xarxa corporativa.

⚠️

Atenció: IP estàtica per al servidor

Un servidor mai ha de tenir IP dinàmica (DHCP): si la IP canvia, els clients que apunten a ell deixen de funcionar. Proxmox s'ha de configurar amb IP estàtica durant la instal·lació. Consulta al professor quina IP estàtica lliure pots usar dins del rang del taller.
