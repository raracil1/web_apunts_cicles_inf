Què és una Base de Dades?

Conjunt organitzat de dades emmagatzemades electrònicament, estructurades per facilitar-ne l'accés, la gestió i l'actualització eficient.

⚙️

SGBD

Sistema Gestor de Bases de Dades: programari que gestiona l'emmagatzematge, l'accés i la seguretat de les dades. LibreOffice Base utilitza el motor HSQLDB.

🔒

Avantatges

Elimina redundàncies, garanteix la integritat de les dades, permet l'accés multiusuari i facilita consultes complexes de manera eficient.

🧩

Conceptes clau

Taula = files + columnes. Registre = una fila. Camp = una columna. Clau = identificador únic.

Anatomia d'una taula
🔑 ID_Alumne (PK)DNINomCognomsCorreuData_Alta

287654321BMariaPérez Ruizmaria@correu.cat2024-09-01
311223344CCarlesMartínez Sanzcarles@correu.cat2024-09-02
💡
Cada fila és un registre. Cada columna és un camp. El camp ID_Alumne és la clau primària: identifica de manera única cada registre i mai no es pot repetir ni ser NULL.

Model Relacional

Les dades s'organitzen en taules que es relacionen entre elles mitjançant claus. Dissenyat per E.F. Codd el 1970, continua sent el model dominant avui dia.

Clau Primària (PRIMARY KEY)

Camp (o conjunt de camps) que identifica de manera única cada fila de la taula.

  • ✅ Valor únic a tota la taula
  • ✅ Mai no pot ser NULL
  • ✅ No hauria de canviar al llarg del temps
  • ✅ L'ideal és que sigui un nombre enter autoincremental (IDENTITY)
Clau Forània (FOREIGN KEY)

Camp en una taula que fa referència a la clau primària d'una altra taula. Estableix la relació entre taules i garanteix la integritat referencial.

Tipus de relacions
1️⃣

Un a Un (1:1)

Un registre d'A es relaciona amb un únic registre de B.

Ex: Persona ↔ Passaport

🔁

Un a Molts (1:N)

Un registre d'A es relaciona amb diversos de B. La FK va a la taula del costat "molts".

Ex: Client → Comandes

🔀

Molts a Molts (N:M)

Es resol amb una taula intermèdia que conté les FK de les dues taules.

Ex: Alumnes ↔ Assignatures

⚠️
Integritat referencial: No pots inserir una FK que no existeixi a la taula referenciada. Tampoc pots eliminar un registre pare si té fills relacionats (tret que facis servir ON DELETE CASCADE).

Restriccions de camp

NOT NULL

El camp no pot quedar buit. Obligatori en camps essencials.

UNIQUE

Tots els valors han de ser diferents. Per a camps com DNI o correu electrònic.

DEFAULT valor

Valor per defecte si no s'especifica. Ex: DEFAULT CURRENT_DATE

IDENTITY

Autoincrement. El motor assigna el número següent automàticament. Ideal per a PKs.

CHECK (condició)

Valida que el valor compleixi la condició. Ex: CHECK (Edat >= 0)

PRIMARY KEY

Combinació de NOT NULL + UNIQUE. Identifica de manera única el registre.

💡
Per a diners fes servir sempre DECIMAL, mai DOUBLE (els decimals aproximats acumulen errors d'arrodoniment). Per a textos de longitud variable, VARCHAR és més eficient que CHAR.

Crear Taules

Dos mètodes: la interfície gràfica de LibreOffice Base o sentències SQL. Tots dos creen el mateix resultat.

Mètode 1 — Vista Disseny (Gràfic)

Obrir la vista disseny

Panell esquerre → Taules → "Crea una taula en la vista Disseny…"

Definir els camps

Per a cada camp: escriu el Nom, selecciona el Tipus i al panell inferior configura la longitud, el valor predeterminat, l'entrada requerida (NOT NULL) i el valor automàtic (IDENTITY).

Establir la Clau Primària

Clic dret sobre la fila del camp → "Clau primària". Apareixerà la icona 🔑.

Desar

Ctrl+S → Assigna un nom a la taula → Accepta.

Mètode 2 — SQL (CREATE TABLE)
SQL-- Taula bàsica de clients
CREATE TABLE CLIENTS (
    ID_Client   INTEGER      IDENTITY PRIMARY KEY,
    DNI         VARCHAR(9)   NOT NULL UNIQUE,
    Nom         VARCHAR(50)  NOT NULL,
    Cognoms     VARCHAR(100) NOT NULL,
    Correu      VARCHAR(100),
    Telefon     VARCHAR(15),
    Actiu       BOOLEAN      DEFAULT TRUE,
    Data_Alta   DATE         DEFAULT CURRENT_DATE
);
SQL-- Taula amb clau forània
CREATE TABLE COMANDES (
    ID_Comanda  INTEGER       IDENTITY PRIMARY KEY,
    ID_Client   INTEGER       NOT NULL,
    Data        DATE          DEFAULT CURRENT_DATE,
    Total       DECIMAL(10,2) NOT NULL,
    Estat       VARCHAR(20)   DEFAULT 'Pendent',
    FOREIGN KEY (ID_Client) REFERENCES CLIENTS(ID_Client)
);
Modificar i eliminar taules
SQL-- Afegir columna
ALTER TABLE CLIENTS ADD COLUMN Adreca VARCHAR(200);

-- Eliminar columna
ALTER TABLE CLIENTS DROP COLUMN Adreca;

-- Canviar tipus (HSQLDB)
ALTER TABLE CLIENTS ALTER COLUMN Correu SET DATA TYPE VARCHAR(150);

-- Eliminar taula completa (IRREVERSIBLE!)
DROP TABLE COMANDES;
DROP TABLE CLIENTS;
⚠️
Per eliminar taules amb FK, primer elimina les taules fill (les que tenen la FK) i després la taula pare.

Inserir Dades

Pots introduir dades directament al full de dades o fent servir la sentència SQL INSERT INTO.

Mètode gràfic

Doble clic sobre la taula al panell de Taules → S'obre el full de dades → Escriu directament a les cel·les → En canviar de fila, les dades es desen automàticament.

💡
Els camps amb IDENTITY s'omplen sols. Els camps amb DEFAULT es poden deixar en blanc i s'aplicarà el valor predeterminat.
INSERT INTO — Sintaxi SQL
SQL-- Un sol registre (especificant columnes)
INSERT INTO CLIENTS (DNI, Nom, Cognoms, Correu)
VALUES ('12345678A', 'Joan', 'García López', 'joan@correu.cat');

-- Diversos registres alhora
INSERT INTO CLIENTS (DNI, Nom, Cognoms, Correu, Telefon)
VALUES
    ('11111111A', 'Anna',  'López Vera',    'anna@correu.cat',  '611000001'),
    ('22222222B', 'Pere',  'Sanz Ruiz',     'pere@correu.cat',  '622000002'),
    ('33333333C', 'Laura', 'Fernández Gil', 'laura@correu.cat', '633000003');

-- Amb FK (ID_Client ha d'existir a CLIENTS)
INSERT INTO COMANDES (ID_Client, Total, Estat)
VALUES (1, 245.50, 'Enviat');

-- Fent servir funcions de data

VALUES (2, 89.99); -- Data = CURRENT_DATE, Estat = 'Pendent' (per defecte)

Consultes SELECT

La sentència més usada en SQL. Permet recuperar, filtrar, ordenar i agrupar dades d'una o diverses taules.

Estructura general
SQLSELECT   camps           -- què mostrar
FROM     taula(es)       -- d'on
WHERE    condició        -- filtre de files
GROUP BY camp           -- agrupar
HAVING   condició       -- filtre de grups
ORDER BY camp [ASC|DESC] -- ordenar
LIMIT    n;             -- limitar resultats
Exemples bàsics
SQL-- Tots els camps
SELECT * FROM CLIENTS;

-- Camps específics
SELECT Nom, Cognoms, Correu FROM CLIENTS;

-- Àlies de columna
SELECT Nom AS 'Nom Client', Correu AS 'Adreça electrònica' FROM CLIENTS;

-- Sense duplicats
SELECT DISTINCT Estat FROM COMANDES;
Filtres amb WHERE
SQL-- Comparació exacta
SELECT * FROM CLIENTS WHERE DNI = '12345678A';

-- Major/menor
SELECT * FROM PRODUCTES WHERE Preu > 50;

-- AND / OR / NOT
SELECT * FROM PRODUCTES WHERE Preu > 50 AND Stock < 20;
SELECT * FROM CLIENTS   WHERE Nom = 'Anna' OR Nom = 'Joan';

-- BETWEEN (rang inclusiu)
SELECT * FROM PRODUCTES WHERE Preu BETWEEN 20 AND 100;

-- IN (llista de valors)
SELECT * FROM COMANDES  WHERE Estat IN ('Pendent', 'En procés');

-- LIKE (patrons): % = diversos caràcters, _ = un sol caràcter
SELECT * FROM CLIENTS   WHERE Nom     LIKE 'J%';        -- comença per J
SELECT * FROM CLIENTS   WHERE Correu  LIKE '%@gmail.com';
SELECT * FROM CLIENTS   WHERE Cognoms LIKE '%García%';

-- IS NULL / IS NOT NULL
SELECT * FROM CLIENTS   WHERE Correu IS NULL;
Ordenar i limitar
SQL-- Ordenar (ASC per defecte)
SELECT * FROM PRODUCTES ORDER BY Preu DESC;
SELECT * FROM CLIENTS   ORDER BY Cognoms ASC, Nom ASC;

-- Top 5 (HSQLDB fa servir TOP n abans del camp, o LIMIT al final)
SELECT TOP 5 * FROM PRODUCTES ORDER BY Preu DESC;
Funcions d'agregació
SQLSELECT COUNT(*)           AS Total_Clients    FROM CLIENTS;
SELECT SUM(Total)          AS Vendes_Totals    FROM COMANDES;
SELECT AVG(Preu)           AS Preu_Mitja       FROM PRODUCTES;
SELECT MAX(Preu), MIN(Preu)               FROM PRODUCTES;

-- GROUP BY: agrupa i aplica funcions per grup
SELECT   ID_Client, COUNT(*) AS Num_Comandes, SUM(Total) AS Total_Gastat
FROM     COMANDES
GROUP BY ID_Client
HAVING   SUM(Total) > 200   -- HAVING filtra GRUPS (no files)
ORDER BY Total_Gastat DESC;
JOIN — Consultes multitaula
SQL-- INNER JOIN: només files amb coincidència a les dues taules
SELECT C.Nom, C.Cognoms, P.Data, P.Total
FROM   COMANDES P
INNER JOIN CLIENTS C ON P.ID_Client = C.ID_Client;

-- LEFT JOIN: tots els clients (tinguin o no comandes)
SELECT C.Nom, C.Cognoms, P.ID_Comanda, P.Total
FROM   CLIENTS C
LEFT JOIN COMANDES P ON C.ID_Client = P.ID_Client;

-- JOIN triple: comanda + client + producte
SELECT C.Nom, P.Data, PR.Nom AS Producte, L.Quantitat
FROM   LINIES_COMANDA L
JOIN   COMANDES  P  ON L.ID_Comanda  = P.ID_Comanda
JOIN   CLIENTS   C  ON P.ID_Client   = C.ID_Client
JOIN   PRODUCTES PR ON L.ID_Producte = PR.ID_Producte;
💡
Fes servir àlies de taula (la lletra darrere del nom) per escurçar les consultes amb JOIN. Ex: FROM COMANDES P — després pots escriure P.Total en lloc de COMANDES.Total.

UPDATE i DELETE

Modificar i eliminar registres amb SQL. Utilitza sempre WHERE per no afectar tota la taula.

UPDATE — Modificar dades
SQL-- Actualitzar un camp d'un registre concret
UPDATE CLIENTS
SET    Correu = 'nou@correu.cat'
WHERE  ID_Client = 1;

-- Actualitzar diversos camps alhora
UPDATE CLIENTS
SET    Nom = 'Maria Josep', Telefon = '699000001'
WHERE  DNI = '22222222B';

-- Actualitzar amb càlcul
UPDATE PRODUCTES
SET    Preu = Preu * 1.05   -- pujada del 5%
WHERE  ID_Categoria = 3;

-- Actualitzar basat en una altra taula (subconsulta)
UPDATE COMANDES
SET    Estat = 'Completat'
WHERE  Data < '2024-01-01' AND Estat = 'Enviat';
⚠️
Un UPDATE sense WHERE modifica tots els registres de la taula. Abans d'executar-lo, prova el WHERE amb un SELECT per comprovar quines files afecta.
DELETE — Eliminar dades
SQL-- Eliminar un registre concret
DELETE FROM CLIENTS WHERE ID_Client = 5;

-- Eliminar diversos amb condició
DELETE FROM COMANDES WHERE Estat = 'Cancel·lat' AND Data < '2023-01-01';

-- Buidar taula (però mantenir l'estructura)
DELETE FROM REGISTRES_SISTEMA;   -- esborra totes les files, la taula segueix existint
⚠️
No pots eliminar un registre pare si té registres fill relacionats (FK). Primer elimina els fills o fes servir ON DELETE CASCADE a la definició de la FK.

Propietats clau d'un control

Clic dret sobre un control → "Propietats del control" (o F4). Propietats més importants:

  • Camp de dades: a quina columna de la taula està vinculat
  • Longitud màxima: nombre màxim de caràcters permesos
  • Entrada requerida: si és Sí, no es pot desar el registre sense aquest valor
  • Només lectura: l'usuari pot veure però no editar (ideal per a IDs)
  • Ordre de tabulació: la seqüència de camps en prémer Tab

Informes

Els informes permeten presentar les dades de manera estructurada i llesta per imprimir o exportar.

Crear un informe amb l'Assistent

Panell Informes

Panell esquerre → Informes → "Utilitza l'assistent per a crear informes…"

Seleccionar l'origen

Tria la taula o consulta que servirà de font de dades. Afegeix els camps que vols mostrar.

Agrupació

Pots agrupar les dades per un camp (p. ex: agrupar comandes per client) per obtenir subtotals.

Criteris d'ordenació

Defineix fins a 4 camps d'ordenació amb la seva direcció (ASC/DESC).

Disseny i títol

Tria l'orientació (vertical/horitzontal), l'estil visual i escriu el títol de l'informe.

Tipus d'informes
📊

Informe estàtic

Genera una "fotografia" de les dades en aquell moment. Es desa com a document ODF.

🔄

Informe dinàmic

S'actualitza automàticament amb les dades actuals cada vegada que s'obre.

💡
Si necessites un informe amb totals o subtotals, crea primer una consulta SQL que calculi aquests valors (amb GROUP BY i SUM) i utilitza aquesta consulta com a font de l'informe.

🎓 Exercici 2 — Centre Educatiu

Mitjà

Dissenya la base de dades d'un centre educatiu amb alumnes, assignatures, professors i matrícula:

  • ALUMNES: ID_Alumne, DNI, Nom, Cognoms, Data_Naixement, Correu
  • ASSIGNATURES: ID_Assignatura, Codi UNIQUE, Nom, Hores, Curs
  • PROFESSORS: ID_Professor, DNI, Nom, Cognoms, Especialitat
  • MATRICULES: ID_Matricula, FK ID_Alumne, FK ID_Assignatura, Any, Nota DECIMAL(4,2)
  • IMPARTEIX: ID_Imparteix, FK ID_Professor, FK ID_Assignatura, Aula, Horari

Consultes a realitzar:

  • Alumnes matriculats a cada assignatura (GROUP BY + COUNT)
  • Nota mitjana de cada alumne (GROUP BY + AVG)
  • Alumnes amb nota > 7 (WHERE + JOIN)
  • Assignatures sense professor assignat (LEFT JOIN + IS NULL)

Repte addicional: Crea un formulari d'ALUMNES amb subformulari de MATRÍCULES i un informe amb les notes de cada alumne agrupat per assignatura.
