DESPLAÇA
Conceptes bàsics

Ofimàtica i aplicacions d'oficina

L'ofimàtica agrupa totes les eines informàtiques que donen suport a les tasques administratives i de gestió d'una oficina o empresa. Instal·lar-les correctament és la base del treball d'un tècnic SMX.

📝

Processadors de text

Permeten crear, editar i formatar documents de text. Exemples: Microsoft Word, LibreOffice Writer, Google Docs. Formats principals: .docx, .odt, .pdf.

📊

Fulls de càlcul

Eina per a organitzar dades numèriques, fer càlculs i generar gràfics. Exemples: Microsoft Excel, LibreOffice Calc, Google Sheets. Formats: .xlsx, .ods, .csv.

📽️

Presentacions

Creació de diapositives per a exposicions i ponències. Exemples: Microsoft PowerPoint, LibreOffice Impress, Google Slides. Format: .pptx, .odp.

📧

Gestors de correu i agenda

Clients de correu electrònic i organitzadors personals. Exemples: Microsoft Outlook, Mozilla Thunderbird, Evolution. Protocols: IMAP, SMTP, CalDAV.

🗄️

Bases de dades d'escriptori

Per a gestionar informació estructurada. Exemples: Microsoft Access, LibreOffice Base. Usos: gestió d'inventaris, clients, proveïdors.

☁️

Suite online (SaaS)

Aplicacions que funcionen des del navegador sense instal·lació local. Exemples: Google Workspace, Microsoft 365 Online, Zoho Office. Avantatge: accés des de qualsevol dispositiu.

📋 Fase 1: Anàlisi de requisits

Abans d'instal·lar qualsevol aplicació, el tècnic ha de verificar:

  • Requisits de maquinari: CPU, RAM, espai en disc, GPU
  • Sistema operatiu compatible: versió de Windows, macOS o distribució Linux
  • Dependències: .NET Framework, Visual C++ Redistributable, Java Runtime
  • Compatibilitat amb el programari existent (conflictes de versions)
Exemple: Microsoft 365 Apps

Requereix: Windows 10/11 (64 bits), 4 GB RAM (8 recomanats), 4 GB d'espai lliure, resolució 1280×768. Necessita connexió a Internet per a activar i sincronitzar.

🔐 Fase 3: Verificació d'integritat

Sempre cal verificar que el fitxer descarregat no ha estat manipulat:

# Verificar hash SHA-256 a Windows PowerShell
Get-FileHash C:\Downloads\LibreOffice_7.6.msi -Algorithm SHA256

# Comparar amb el hash oficial de la pàgina web
# Si coincideix → fitxer íntegre ✅
# Si no coincideix → no instal·lar ❌
⚠️

Descarregar programari de fonts no oficials és el principal vector d'entrada de programari maliciós (malware) a les empreses.

Mètodes d'instal·lació

Tipus d'instal·lació segons l'entorn

En funció de l'entorn (ordinador personal, empresa petita o gran corporació), el mètode d'instal·lació varia significativament.

🖱️

Instal·lació assistida (GUI)

L'assistent gràfic guia l'usuari pas a pas. És el mètode més comú per a usuaris individuals. Permet triar la carpeta de destinació, els components a instal·lar i les opcions de configuració bàsiques.

Senzilla Lenta per a molts equips Windows .msi / .exe
⌨️

Instal·lació desatesa (Silent)

S'executa sense intervenció de l'usuari, amb paràmetres predefinits. Ideal per a desplegar en múltiples equips simultàniament. Es fa amb scripts de PowerShell, Batch o gestors de paquets.

# Exemple: instal·lació silenciosa de LibreOffice
msiexec /i LibreOffice_7.6.msi /quiet /norestart
# /quiet = sense GUI /norestart = no reinicia
📦

Gestors de paquets

Eines que automatitzen la descàrrega, instal·lació, actualització i desinstal·lació d'aplicacions des de repositoris centralitzats.

# Windows: Winget
winget install LibreOffice.LibreOffice
winget upgrade --all

# Linux (Ubuntu/Debian): APT
sudo apt update && sudo apt install libreoffice
🌐

Desplegament remot (MDM/GPO)

En entorns empresarials, s'utilitzen eines de gestió centralitzada per desplegar aplicacions a tots els equips del domini sense anar físicament a cada ordinador.

Eines corporatives

Microsoft Intune, SCCM/MECM, Group Policy (GPO), Ansible (Linux). Permeten controlar quines versions tenen tots els equips de l'empresa.

Llicències de programari

Tipus de llicències i models de distribució

Entendre les llicències és essencial per a un tècnic SMX. Instal·lar programari sense llicència vàlida és il·legal i pot comportar sancions greus per a l'empresa.

Propietari

Programari privatiu / propietari

El codi font no és accessible. L'empresa desenvolupa i ven el dret d'ús. La llicència defineix quantes instal·lacions, per quant de temps i en quines condicions es pot usar.

Exemples

Microsoft Office (LTSC o 365), Adobe Acrobat Pro, AutoCAD, Antivirus comercials. Requereixen clau de producte o compte vinculat.

Lliure

Programari lliure i de codi obert

El codi font és accessible i modificable. Les llicències lliures (GPL, MIT, Apache, LGPL) permeten usar, estudiar, modificar i redistribuir el programari lliurement.

Exemples

LibreOffice (llicència MPL 2.0), Mozilla Thunderbird (MPL), VLC (GPL), GIMP (GPL). Gratuïts i sense restriccions de còpies.

Freeware

Freeware / Freemium

Gratuït per a l'ús, però el codi font no és accessible. El Freeware no permet modificar ni redistribuir el programari. El Freemium ofereix una versió bàsica gratuïta i funcionalitats avançades de pagament.

Exemples

Adobe Reader (freeware), Zoom (freemium), Notion (freemium), Slack (freemium). Legals per a ús professional en la versió gratuïta.

SaaS

SaaS — Programari com a servei

El programari no s'instal·la localment: s'accedeix per Internet. Es paga una subscripció mensual o anual per empresa o per usuari. El proveïdor s'encarrega de les actualitzacions i el manteniment.

Exemples

Google Workspace (6–18 €/mes per usuari), Microsoft 365 Business (6–22 €/mes per usuari), Salesforce. El rol del tècnic SMX és gestionar els comptes i permisos.

⚖️

Marc legal: Llei de Propietat Intel·lectual

A Espanya, instal·lar programari sense llicència vàlida (pirateria) viola el Text Refós de la Llei de Propietat Intel·lectual (TRLPI). Les sancions van des de multes de fins a 600.000 € fins a penes de presó de fins a 4 anys en casos greus. Com a tècnic SMX, tens la responsabilitat de garantir que tota instal·lació és legalment conforme.

Actualitzacions i manteniment

Actualització i manteniment del programari

Mantenir el programari actualitzat és una tasca fonamental del tècnic SMX. Les actualitzacions no solament afegeixen funcions: corregeixen vulnerabilitats de seguretat crítiques.

🛡️

Pedaços de seguretat

Corregeixen vulnerabilitats descobertes en el programari. Han d'aplicar-se amb prioritat màxima. Microsoft publica pedaços els dimarts de cada mes (Patch Tuesday). LibreOffice publica actualitzacions de seguretat de forma independent.

🔧

Actualitzacions de funcions

Afegeixen noves funcionalitats o milloren el rendiment. No sempre cal aplicar-les immediatament: és recomanable esperar que la comunitat detecti possibles problemes (errors de regressió).

🔬

Versions LTS i estables

Les versions Long Term Support (LTS) ofereixen manteniment garantit durant varis anys. Ideals per a entorns empresarials on l'estabilitat és prioritat. LibreOffice té versions "still" (LTS) i "fresh" (novetats).

Procediment d'actualització segura

  • Còpia de seguretat prèvia: Abans d'actualitzar, fes un punt de restauració del sistema (Windows) o una còpia de seguretat de les dades importants. Si l'actualització falla, podràs revertir els canvis.

  • Consulta les notes de la versió (Release Notes): Llegeix el changelog per saber quins errors es corregeixen i quines noves funcions s'incorporen. Verifica si hi ha incompatibilitats conegudes amb el teu sistema.

  • Prova en un equip pilot: En entorns corporatius, aplica primer l'actualització en un equip de prova i verifica durant 24-48 hores que tot funciona correctament.

  • Desplegament gradual: Actualitza primer els equips menys crítics. Si no hi ha problemes, desplega progressivament a la resta de l'organització.

  • Verificació post-actualització: Comprova que l'aplicació s'inicia correctament, que els fitxers existents s'obren sense errors i que les macros i complements (add-ins) segueixen funcionant.

  • Documentació: Registra la versió instal·lada, la data i l'equip en el full de control d'inventari de programari de l'empresa.

Desinstal·lació

Desinstal·lació neta de programari

Desinstal·lar correctament és tan important com instal·lar. Una desinstal·lació incompleta pot deixar restes al registre del sistema i ralentir l'equip.

🗑️ Mètodes de desinstal·lació

  • Panel de control / Configuració: Mètode estàndard de Windows. Adequat per a la majoria de casos.
  • Desinstal·lador propi: Alguns programes inclouen el seu propi uninstaller (ex: unins000.exe).
  • Winget: winget uninstall LibreOffice.LibreOffice
  • Revo Uninstaller / IObit: Eines de tercers que eliminen residus del registre.
  • APT (Linux): sudo apt remove --purge libreoffice*

🧹 Neteja post-desinstal·lació

Després de desinstal·lar, cal verificar i netejar:

# Carpetes residuals típiques a Windows
C:\Program Files\[Aplicació]
C:\Users\[Usuari]\AppData\Local\[App]
C:\Users\[Usuari]\AppData\Roaming\[App]

# Claus de registre (amb precaució)
HKEY_LOCAL_MACHINE\SOFTWARE\[Vendor]
HKEY_CURRENT_USER\SOFTWARE\[Vendor]
Inventari i gestió

Inventari de programari instal·lat

En un entorn professional, el tècnic SMX ha de mantenir un registre actualitzat de tot el programari instal·lat a l'empresa per garantir el compliment de llicències i facilitar el suport.

📋 Llistat de programari per línia de comandes

# Windows PowerShell: llistat complet
Get-ItemProperty HKLM:\Software\Microsoft\Windows\CurrentVersion\Uninstall\* | Select-Object DisplayName, DisplayVersion, Publisher | Export-Csv inventari.csv

# O amb Winget
winget list > inventari.txt

# Linux: paquets instal·lats
dpkg --get-selections | grep -v deinstall > inventari.txt

🏢 Connexió amb el món real

Aquest és exactament el primer encàrrec real d'un tècnic SMX júnior quan entra a una empresa: "tenim programari il·legal, regularitza-ho". La migració a programari lliure o SaaS, la documentació de llicències i l'automatització d'actualitzacions via scripts no és un exercici acadèmic —és el dia a dia en pimes i administracions públiques. La Llei de Propietat Intel·lectual responsabilitza el tècnic que realitza les instal·lacions, no només l'empresa.

🏢

Escenari

Ets el nou tècnic SMX de l'empresa InnovaGest S.L., una assessoria fiscal amb 15 empleats. L'empresa usa ordinadors amb Windows 11. El responsable d'IT t'encarrega migrar dels programes ofimàtics actuals (versions antigues sense llicència vàlida) a una solució legal, actualitzada i ben documentada. Disposes d'un pressupost limitat i has de plantejar una proposta amb programari lliure + SaaS.

📁 PART 1 — Anàlisi de l'entorn i proposta de solució

  • Inventari inicial: Usa PowerShell (winget list o el cmdlet Get-ItemProperty) per obtenir el llistat complet del programari instal·lat en el teu equip (o en una màquina virtual). Exporta-ho a un fitxer CSV. Identifica quines aplicacions ofimàtiques estan presents i quina versió tenen.

  • Anàlisi de llicències: Per a cada aplicació ofimàtica trobada, classifica-la com: llicència vàlida, llicència caducada, sense llicència (programari il·legal) o programari lliure. Elabora un informe en format taula (pots usar un full de càlcul LibreOffice Calc).

  • Proposta de migració: Redacta un document de proposta (LibreOffice Writer) que inclogui: (a) comparativa LibreOffice vs Microsoft 365 (funcionalitats, cost, compatibilitat); (b) recomanació justificada; (c) pla de migració en 3 fases; (d) estimació del temps necessari.

📦 PART 2 — Instal·lació desatesa i verificació

  • Descàrrega verificada: Descarrega l'última versió estable de LibreOffice des de libreoffice.org. Verifica la integritat del fitxer comparant el hash SHA-256 oficial amb el que obté PowerShell. Documenta el resultat amb una captura de pantalla.

  • Instal·lació silenciosa: Executa la instal·lació de LibreOffice amb paràmetres silenciosos des de la línia de comandes. Documenta el comandament complet usat. Verifica que la instal·lació s'ha completat correctament comprovant la versió: soffice --version.

  • Configuració post-instal·lació: Configura LibreOffice perquè: (a) guardi sempre els documents en format .docx i .xlsx per defecte (compatibilitat amb MS Office); (b) tingui activades les actualitzacions automàtiques; (c) inclogui la signatura corporativa a Writer.

  • Instal·lació del paquet d'idioma català: Descarrega i instal·la el paquet d'idioma i corrector ortogràfic en català per a LibreOffice. Verifica que la correcció ortogràfica en català funciona correctament.

🔄 PART 3 — Gestió d'actualitzacions i scripting

  • Script d'actualització automatitzada: Crea un script de PowerShell (actualitza_ofimatica.ps1) que: comprovi si hi ha una versió nova de LibreOffice disponible a Winget; si és el cas, la descarregui i instal·li de forma silenciosa; escrigui un log amb data, versió anterior, versió nova i resultat de la operació. El log s'ha de desar a C:\Logs\ActualizacionesSoftware.txt.

  • Tasca programada: Configura el Programador de tasques de Windows perquè executi el script cada dilluns a les 06:00 AM (abans que arribin els empleats). Documenta la configuració amb captures de pantalla.

  • Prova de regressió: Després d'una actualització simulada, obre un document .docx complex (amb taules, imatges i estils) i verifica que s'obre correctament. Documenta qualsevol diferència de format o comportament.

📊 PART 4 — Documentació final i presentació

  • Memòria tècnica: Redacta una memòria de 3–5 pàgines en LibreOffice Writer amb: portada corporativa, índex de continguts automàtic, descripció del procés seguit, problemes trobats i solucions aplicades, conclusions. Usa estils de paràgraf (Títol 1, Títol 2, Normal) per generar l'índex automàticament.

  • Full de control d'inventari final: Crea un full de càlcul LibreOffice Calc amb l'inventari actualitzat de tot el programari instal·lat. Afegeix format condicional perquè: les llicències lliures surtin en verd; les llicències de subscripció en blau; les versions desactualitzades en taronja; el programari no llicenciat en vermell.

  • Presentació: Crea una presentació de 8–10 diapositives a LibreOffice Impress que resumeixi el treball fet per presentar al responsable d'IT. Ha d'incloure: resum executiu, comparativa de costos (gràfic de barres), pla de migració (línia temporal) i conclusions.

📋

Lliurament i criteris d'avaluació

Lliura un fitxer ZIP amb: (1) l'script PowerShell, (2) el log generat, (3) la memòria tècnica en .odt i .pdf, (4) el full de control d'inventari, (5) la presentació .odp.

Criteris: Funcionalitat de l'script (30%), qualitat de la documentació (30%), correcció del procés d'instal·lació (25%), presentació i claredat (15%).

Errors habituals

⚠️ Trampes freqüents del RA1

Errors que fan quasi tots els alumnes la primera vegada. Llegeix-los ara per no caure-hi.

⚠️

No verificar el hash SHA-256 del fitxer descarregat

Descarregar un instal·lador i executar-lo directament sense comprovar la integritat és un risc real. Un fitxer manipulat (atac man-in-the-middle o mirall compromès) pot contenir malware. L'error típic és pensar "ho he descarregat de la pàgina oficial, no cal comprovar-ho". La pàgina oficial pot estar compromesa o la CDN pot servir fitxers incorrectes.

✅ Solució: Sempre executa Get-FileHash fitxer.msi -Algorithm SHA256 a PowerShell i compara el resultat amb el hash publicat a la web oficial. Si no coincideix, no instal·lis el fitxer i reporta-ho.
⚠️

Confondre llicència "Freeware" amb "Programari lliure"

Freeware vol dir gratuït però el codi font és tancat i no pots modificar-lo ni redistribuir-lo. Programari lliure (GPL, MIT...) vol dir que pots veure, modificar i redistribuir el codi. Molts alumnes (i tècnics!) posen Adobe Reader i LibreOffice al mateix sac de "programari gratuït" sense entendre que els seus drets d'ús són completament diferents.

✅ Solució: Recorda la regla: Freeware = gratuït però tancat. Lliure = pots fer-ho tot (usar, estudiar, modificar, redistribuir). A l'inventari d'empresa, el tipus de llicència ha de ser exacte, no "gratuït".
⚠️

Usar la instal·lació gràfica (GUI) per desplegar en molts equips

Instal·lar un programa a 50 equips fent clic a "Següent, Següent, Accepto, Instal·la" en cada un porta moltes hores i és font d'errors humans. L'error és no conèixer els paràmetres d'instal·lació silenciosa. Per exemple, LibreOffice accepta /quiet /norestart.

✅ Solució: Per a qualsevol desplegament de més de 3 equips, usa instal·lació silenciosa via Winget o l'instal·lador amb paràmetres /quiet. Per a 20+ equips, considera Winget + PowerShell o GPO de domini.
⚠️

Aplicar actualitzacions de funcions immediatament en producció

Quan surt una versió nova de LibreOffice, la temptació és aplicar-la immediatament a tots els equips. Les actualitzacions de funcions sovint introdueixen errors de regressió que afecten macros, complements o compatibilitat de fitxers.

✅ Solució: Pedaços de seguretat → aplica immediatament. Actualitzacions de funcions → espera 1–2 setmanes, aplica primer en 1 equip pilot, verifica, i desplega progressivament.
⚠️

No netejar les restes després de desinstal·lar

Desinstal·lar des del "Panel de control" no elimina tot. Queden carpetes a AppData\Local, AppData\Roaming i claus al Registre. Amb el temps, les desinstal·lacions incompletes acumulen gigabytes de residus i poden causar conflictes si re-instal·les la mateixa aplicació.

✅ Solució: Comprova manualment C:\Program Files, %AppData% i %LocalAppData%. Per a una neteja completa, usa Revo Uninstaller (mode avançat) que analitza el registre abans i després.
⚠️

No documentar les instal·lacions a l'inventari

Si l'empresa té 30 instal·lacions d'un software del qual ha pagat 20 llicències, la multa pot ser molt superior al cost de les 10 llicències que falten. El tècnic SMX que va fer les instal·lacions és corresponsable si no va documentar i alertar.

✅ Solució: Cada instal·lació ha de quedar registrada immediatament a l'inventari: nom, versió, equip, data, tipus de llicència i responsable. Automatitza-ho amb PowerShell: winget list | Export-Csv programat setmanalment.

🔐 Verificació SHA-256

❌ Malament

Descarrego LibreOffice de Google, faig doble clic a l'instal·lador i el instal·lo directament sense comprovar res.

Risc: No sé si el fitxer és l'original. Podria contenir malware.

✅ Bé

Descarrego de libreoffice.org oficial. Executo:
Get-FileHash LibreOffice_7.6.msi -Algorithm SHA256
Comparo amb el hash de la web oficial. Coincideix → instal·lo.

Garantia: el fitxer és exactament l'original del fabricant.

🤖 Script d'actualització

❌ Malament — sense log ni control d'errors

winget upgrade LibreOffice.LibreOfficeSi falla, no ho sabràs. No hi ha log, ni versió anterior, ni data. Inútil per a producció.

$log = "C:\Logs\actualitzacions.txt"
$versioAnterior = winget list LibreOffice

Add-Content $log "$(Get-Date) | $versioAnterior → $versioNova | $resultat"Traçabilitat completa de cada actualització.

Avaluació

📊 Rúbrica de correcció — RA1

Llegeix la rúbrica abans de lliurar.

Criteri Pes 🟢 Excel·lent (9–10) 🔵 Bé (7–8) 🟡 Suficient (5–6) 🔴 Insuficient (0–4)
Script PowerShell funcional 30% Script executa correctament, comprova versió, fa instal·lació silenciosa, genera log amb data/versió/resultat. Tasca programada configurada. Script funcional amb log però amb 1–2 errors menors (ruta incorrecta o falta versió anterior). Script parcialment funcional: fa la instal·lació però el log és incomplet o la tasca programada no s'ha configurat. Script no funciona, no genera log o no s'ha creat. Instal·lació feta manualment.
Qualitat de la documentació 30% Memòria 3–5 pàgines amb portada, TOC automàtic, estils Títol 1/2/Normal. Format professional. Inventari amb format condicional correcte. Documentació completa però índex manual, o estils inconsistents, o format condicional parcial. Documentació present però sense TOC automàtic, o sense portada, o inventari sense format condicional. Documentació mínima, incompleta o sense estructura tècnica.
Correcció del procés d'instal·lació 25% Descàrrega oficial, hash SHA-256 verificat i documentat, instal·lació silenciosa correcta, configuració post-instal·lació completa (format .docx, idioma català, signatura). Procés correcte però falta la verificació del hash o la configuració post-instal·lació és parcial. Instal·lació feta però sense verificació hash i configuració post-instal·lació mínima. Instal·lació des de font no oficial, o assistida sense paràmetres silenciosos, o sense captures d'evidència.
Presentació i claredat 15% Presentació Impress 8–10 diapositives amb gràfic de costos, línia temporal i resum executiu. Disseny professional. Presentació completa però sense gràfic de costos o sense línia temporal. Menys de 8 diapositives o sense estructura clara. No lliurada, o menys de 5 diapositives, o tot text sense cap visual.
TOTAL 100% · Nota final sobre 10
💡

On es perd més nota: El 30% de l'script és el criteri amb més variació. Un script que "quasi funciona" però no genera log correctament no és el mateix que un que funciona perfectament. Prova l'script en una màquina virtual neta abans de lliurar.
